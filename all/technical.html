2026: a digital odyssey — 심연의 방정식

작품별 기술 해설 · TECHNICAL SPECIFICATIONS

■ 공통 기술 스택

렌더링
Three.js 0.160.0 (01~14번, jsDelivr CDN 로드) / 15번은 라이브러리 없는 순수 WebGL 1.0
형식
각 작품은 외부 의존성이 거의 없는 단일 HTML 파일 (셰이더·기하·로직 모두 인라인)
최적화
LOWP(저전력) 자동 감지 — 모바일 UA, 화면폭 768px 이하, CPU 코어 4개 이하, 메모리 4GB 이하 중 하나라도 해당하면 파티클 수 감소, 안티앨리어싱 해제, pixelRatio 1로 고정
공통 장치
백그라운드 탭에서 rAF 큐잉(전력 절약), 화면 회전/뷰포트 변화 재동기화(vp-resync), CDN 로드 실패 감지 워치독(7초 후 재시도 안내), 세션 기반 UNIVERSE # 시드, 방문 기록(sessionStorage), 키오스크 무인 모드(90초 무입력 시 관측소 복귀), bfcache/pageshow 대응
사운드
Web Audio API — 08번(마이크 FFT 분석), 14번(절차적 합성: 화이트노이즈+드론+스월), 15번(합성+마이크 분석)

01RGB GALAXY ENGINE

파티클
250,000개 (LOWP/모바일 60,000개), THREE.Points
은하 생성
r = √i × 1.6, θ = i × 0.013 + arm × 2π/3 — 페르마 나선 3개를 120°씩 회전 배치. 거듭제곱 난수(pow 2.2)로 팔 주변 난류 산포, 반지름에 비례한 수직 두께
셰이더
커스텀 ShaderMaterial — 정점별 시드/팔 번호 속성으로 RGB 색상환(cos 팔레트)을 시간에 따라 순환
후처리
WebGLRenderTarget 1패스 → 직접 작성한 GLSL 블룸(6방향 글로우 샘플링 + 임계값), 방사형 색수차(chroma offset), 비네트, 감마 보정(pow 0.85)
블렌딩
AdditiveBlending + depthWrite off + FogExp2

02HYPERBOLIC NEBULA

파티클
55,000개 (LOWP 25,000개)
형태 수학
z = sinh(0.6u)·0.12·cosh(0.45v) − 1.4·tanh(r) — 쌍곡함수로 높이장을 만들고 θ 기반 나선 위상(spiral = 3θ + 1.1r)으로 비틀어 배치. 색은 z·r·θ 각각의 사인파로 RGB 매핑
가스 구름
반지름 60의 구체 내벽에 ShaderMaterial — 해시 기반 3D value noise 3옥타브 FBM, 시간 애니메이션, pow(n,3)로 희박화한 알파
카메라
포인터 위치로 lerp(0.03) 추적하는 패럴랙스

03RIEMANN DREAMS

파티클
60,000개 (LOWP 25,000개)
형태 수학
소수 300개를 순환 인덱스로 사용, θ = 0.045i + 2sin(p), r = 0.55√i + 0.08p — 소수가 나선의 위상을 교란하는 '소수 바다(Prime Ocean)' 격자
애니메이션
CPU에서 매 프레임 위치 버퍼 갱신 — 반지름 파동 sin(0.08r − 2.5t)과 소수 파동 cos(0.4p + 1.5t)의 합성
상호작용
클릭 → 카메라 광선을 y=0 평면에 역투영(unproject)해 해당 좌표에 파문 트리거 / DeviceOrientation API로 모바일 자이로 시차
파문
triggerBloom(x,z) — 클릭 지점 중심의 감쇠 파동을 높이장에 가산

04QUANTUM SINGULARITY

파티클
성운 400,000개 (LOWP 150,000개) + 소수 나선 3,000개 + 배경 별
형태 수학
소수 p마다 θ = p·2π/137.507 (황금각 배치), r = √p × 12 — 해바라기 씨앗 패턴의 소수 필로택시스. h = sin(0.8·해시)로 반지름 요동
진화 시스템
1,800초(30분) 주기 상태기계 — 진행률에 따라 ERA OF GENESIS(~20%) → RIEMANN CHRONICLES(~45%) → HYPERBOLIC CATHEDRAL(~70%) → QUANTUM SINGULARITY. DOM 계기판(evo-phase, prime-factor)을 0.5초마다 갱신
셰이더
성운용 + 중심 블랙홀용 ShaderMaterial 2종, masterGroup 단위 회전

05DUAL GALAXY

파티클
120,000개 (모바일 60,000개) — 두 은하에 절반씩 배분
형태 수학
r = √i × 1.2, θ = 0.015i 나선 2개를 서로 다른 중심·색상 팔레트로 배치, 별도의 배경 별 Points
연출
카메라가 (500cos 0.2t, 120sin 0.13t, 500sin 0.2t) 리사주 궤도로 자율 항해, FogExp2 색상도 RGB 사인파로 시간 순환 — '안개마저 작품의 일부'
중심체
블랙홀 Mesh + 가산혼합 글로우

06LAMBERT NEBULA: ESSENTIAL

파티클
95,000개 (모바일 45,000개) + 배경 별
형태 수학
람베르트 W 함수 주가지 W₀(x) 곡선을 반지름-각도 공간에 매핑, spread 계수로 방사 산포
안개
FogExp2(density 0.0025) — 제목에 명시된 만큼 짙은 지수 안개가 형태의 경계를 지움
카메라

07LAMBERT NEBULA: ARCHIVE

파티클
120,000개 (LOWP 50,000개)
형태 수학
W₀와 W₋₁ 두 가지(branch)의 나선을 서로 다른 중심(center[])에서 겹쳐 생성 — t 매개변수 나선 X = cx + cos(t)·spiral
코어

카메라
반지름 260 원 궤도 자율 공전 + 포인터 표류

08LAMBERT NEBULA (오디오 리액티브)

파티클
60,000개 (LOWP 30,000개), 원본 위치 백업 배열(originalPositions)을 기준으로 변위 계산
오디오
getUserMedia(마이크) → AudioContext + AnalyserNode FFT — 저역(bass)/고역(high) 대역 에너지를 분리 추출해 성운 전체 변형(저역)과 입자 지터(고역)에 각각 매핑. 데이터 모니터(val-bass/high/vis)에 실시간 수치 표시

UI

09BESSEL BLOOM

파티클
12,000개 (LOWP 7,000개) — 평면 격자형 수면
형태 수학
베셀 함수 1종 J₀를 점근 근사 J₀(x) ≈ √(2/πx)·cos(x−π/4) 로 직접 구현. 각 파원까지의 거리 dₙ에 대해 z = Σ J₀(0.7dₙ − 3t)·amp — 다중 파원 중첩 간섭
상호작용
pointerdown → 화면 좌표를 월드로 변환해 파원 추가 + 링 이펙트(RingGeometry, 가산혼합) 스폰. 파원 진폭은 시간 감쇠(Sources Fade)
특징
GPU 셰이더가 아닌 CPU 높이장 갱신 — 파원 배열을 자유롭게 넣고 빼는 유연함을 택한 설계

10LORENZ BUTTERFLY

파티클
4,000개 (LOWP 2,500개) — 각 입자가 독립적인 초기값
수치 적분
로렌츠 계 ẋ=σ(y−x), ẏ=x(ρ−z)−y, ż=xy−βz (σ=10, ρ=28, β=8/3)를 매 프레임 오일러 적분 — 4,000개의 궤적이 동시에 끌개를 그림
상호작용
클릭 → ρ를 36으로 점프시켜 섭동, 이후 매 프레임 ρ += (28−ρ)·0.01 로 지수 복원 — 카오스의 민감성과 끌개의 안정성을 동시에 시연
표현
잔상형 반투명 렌더링 + 가산혼합으로 궤적의 밀도가 곧 밝기가 되는 구조

파티클

광자 고리

12ODYSSEY: ARCHIVE

파티클
420,000개 (LOWP 140,000개) — 전시 최다 파티클
물리 모델
슈바르츠실트 반지름 Rs = 2GM/c² 개념 골격 위에 커 좌표계 끌림 Δθ ∝ exp(−r/450) — 중심에 가까울수록 방위각이 추가 회전(frame dragging)
구조물
광자 고리 Torus(155, 8) + 아인슈타인 링 Torus(260, 2, 반투명 청색) — 중력렌즈 허상의 기하학적 재현
카메라
자율 공전 + 포인터 표류 (11번과 동일 문법으로 연작 구성)

13EVENT HORIZON

파티클
강착원반 150,000개 (LOWP 50,000개) — ISCO(최내곽 안정궤도)부터 외곽까지 반지름별 온도 그라데이션(diskColor)
물리 모델
케플러 각속도 ω ∝ r^(−3/2) (KEPLER_VERTEX 셰이더), 중력 시간 지연 dτ = √(1 − Rs/r)·dt 를 실시간 계산해 계기판(R/Rs, dτ/dt)에 표시
항해 로직
카메라 거리 r의 증감으로 PHASE 판정 — dr<0이면 APPROACH, 근접도>0.78이면 HORIZON SKIM, 그 외 DEPARTURE. 지평선 스침 시 플래시 오버레이
후처리
풀스크린 post ShaderMaterial — 중력렌즈풍 화면 왜곡
조작
pointerdown+move 드래그 시선(isTargeting)

14BLACK HOLE VOYAGE

규모
단일 HTML 1,200여 줄 — 전시 최대 작품. ShaderMaterial 4종(성운/강착원반/렌즈/포스트)
항해 구조
CYCLE=180초 무한 루프, PHASE_BOUNDS 10단계 [0,22,40,58,78,96,110,145,162,180]. 각 단계마다 영문 서사 + 한글 문구 + 수식 라벨(math-label) 전환
핵심 연출
좌표계 끌림 — 셰이더에서 θ += uFrameDrag·e^(−r/260)·t / 아인슈타인 링 — mix(Colorstd, Colorein, uEinstein) / 슬릿-스캔(108–138초) — smoothstep 강도로 시간 축 파편화 / 대붕괴(145–180초) — collapseIntensity 봉우리 함수
사운드
Web Audio 절차 합성 — 화이트노이즈 버퍼(2초 루프) + 저주파 드론(45→28Hz) + 스월. 하강 진행률에 따라 주파수/음량을 setTargetAtTime으로 연속 보간, 붕괴 시 전체 게인 소멸
계기판
DESCENT %(145초 기준), R/Rs(152초 후 0.000 고정), 한글 악장 문구 페이드 전환
최적화
이 작품만 LOWP 대신 자체 isMobile 분기 사용

15KERR PLASMA CATHEDRAL (순수 WebGL)

렌더링
Three.js 없이 WebGL 1.0 직접 구동 — 풀스크린 삼각형 2개(drawArrays TRIANGLES, 6정점)에 프래그먼트 셰이더 하나로 전체 장면을 레이마칭
물리 모델
커 계량(Kerr metric) ds² 전개식을 제목에 명시 — 셰이더 내부에서 사건의 지평선 rE = M(1+√(1−a²cos²θ)) (외부 지평선/에르고스피어)을 계산해 막(membrane) 발광으로 시각화. 볼류메트릭 FBM 플라스마 밀도 exp(−|r−2.1|·3)·fbm, 투과율 누적(transmittance)
유니폼
u_spin(드래그 스핀), u_bass/u_treble(오디오 대역), u_wake(관측 상태), u_fin(피날레 진행률), mouse, time
관측/수면
마지막 입력 시각 기반 wake 감쇠 — u_wake가 0에 가까워지면 색이 잿불(ember)로 식고 별 레이어가 떠오름. 입력 시 재점화. 색수차도 wake에 비례
오디오
절차 합성(오실레이터+필터, FIN 시 필터 컷오프 140→5,340Hz 상승) + 마이크 입력 FFT(bass/treble 추출)
피날레
FIN 클릭 → u_fin 0→1 진행(FIN_DUR), 질량 손실 mLoss = 1−0.9·uFin으로 플라스마 소산 → 완료 시 코다(시네마틱)/관측소(아카이브)로 자동 이동
